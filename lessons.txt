<h1 align="center"><font family="CenturyGothic" color="#FFFFFF"><strong><u>XHTML</u></strong></font></h1>
<p><font color="#FFFFFF"> XHTML is an acronym for eXtensible HyperText Markup Language.<br><br>
The ”x” came from XML or eXtensible Mark-up Language.<br><br>
Though different from HTML, XHTML also uses the filename extension .html<br><br>
The content of the actual coding might change because of XHTML conventions but it still holds the same filename extension.<br><br><br></font></p><br><br>
<h3><font color="#FFFFFF"><strong>Proper Nesting</strong></font></h3>
<p><font color="#FFFFFF">Proper nesting of elements has been exemplified and XHTML elements must be properly nested.  Proper nesting of elements would mean that, for example, if you were to make a piece of text appear in boldface and italics using <b></b> and <i></i> respectively.</font></p><br>
<h3><font color="#FFFFFF"><strong>Proper Tag Closing</strong></font></h3>
<p><font color="#FFFFFF">Recall the container tags.  Container tags have start and end tags; and end tags are defined by a slash.  End tags are used to basically close the tag.  XHTML requires proper closing of tags.<br>Recall empty tags.  Empty tags are stand-alone type of tags, they don’t require an end tag.  But since XHTML requires proper closing, things have changed a bit for empty tags.  A slash defines end tags, thus, it simplifies closing.  In the same sense, a slash will be used in empty tags for it to be properly closed.  A slash is placed before the end of an angle bracket of a tag.</font></p><br>
<h3><font color="#FFFFFF"><strong>Elements in lowercase letters</strong></font></h3>
<p><font color="#FFFFFF">XHTML elements MUST be written in lowercase characters.</font></p><br>
<h3><font color="#FFFFFF"><strong>One Root Element</strong></font></h3>
<p><font color="#FFFFFF">All XHTML elements MUST be nested within one root element which is html /html.</font></p><br>
<h3><font color="#FFFFFF"><strong>Attributes in lowercase letters</strong></font></h3>
<p><font color="#FFFFFF">All XHTML elements MUST be written in lowercase characters same as with the attributes.</font></p><br>
<h3><font color="#FFFFFF"><strong>Quoted values</strong></font></h3>
<p><font color="#FFFFFF">All XHTML elements MUST be written in lowercase characters same as with the attributes.</font></p><br>
<h3><font color="#FFFFFF"><strong>Forbidden Attribute Minimization</strong></font></h3>
<p><font color="#FFFFFF">In HTML, there are attributes that don’t need values; just like the noshade attribute of horizontal rules, noresize attributes of frames and checked attribute of checkboxes and radio buttons.  But XHTML forbids such (also known as attribute minimization).  In order to fix this, XHTML requires that type of attributes to have a value equal to its attribute name.</font></p><br>
<h3><font color="#FFFFFF"><strong>Image Alternate Texts</strong></font></h3>
<p><font color="#FFFFFF">Remember the alt attribute in images?  It is used to display an alternate text of the text that appears when the mouse hovers over the image and when the image cannot be displayed.  In XHTML, image alternative texts are now mandatory.  In cases where the image is only for decoration purposes, you can provide it a null value by using a space.</font></p><br>
<h3><font color="#FFFFFF"><strong>Amperstands in URLs</strong></font></h3>
<p><font color="#FFFFFF">Amperstands are usually used in URLs to carry variables.  In XHTML, it is required to escape this using the special character entity (&amp).  On Web browser screens, it is of course displayed as amperstands.</font></p><br>
<h3><font color="#FFFFFF"><strong>Contents within Block-level elements</strong></font></h3>
<p><font color="#FFFFFF">In XHTML Strict, you cannot just put content directly inside <body></body>.Contents must be placed within p /p, ol /ol, etc.  (also known as block-level elements).  Another good reason to put <span></span> and <div></div> into practice.</font></p><br><br><br>
